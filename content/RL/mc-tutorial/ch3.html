第 3 章告别上帝视角：MC ε-Greedy 与探索-利用

探索性出发要求把智能体「空投」到任意状态-动作对上重启。模拟器里可以，现实里不行—— 你不能把一台正在学走路的机器人随时抓起来摆到任意姿态。这一章去掉这个假设，代价是直面强化学习最根本的矛盾： 探索与利用（exploration vs. exploitation）。

3.1问题出在「贪婪」上

第 2 章末尾我们看到：探索性出发之所以必要，是因为贪婪策略是确定性的—— 它在每个状态只走一个动作，别的动作永远没有出场机会，估错了也无人翻案。 既然病根是「确定性」，药方就显而易见：让策略松一点口，每个动作都保留一丝被选中的概率。 这样的策略叫软性策略（soft policy）。 只要策略是软的，一条足够长的轨迹自己就能跑遍所有 (s,a)——探索不再依赖「空投」，而是策略的内生能力。

3.2ε-greedy：在贪婪上撕开一道口子

最常用的软性策略是 ε-greedy。设动作共 |𝒜| 个（本章网格 |𝒜|=5），取一个 ε ∈ [0,1]：

π(a|s) = 1 − ε + ε/|𝒜|  （贪婪动作）；  π(a|s) = ε/|𝒜|  （其余每个动作） (3.1)

直观读法：先抽一个 [0,1] 均匀随机数，大于 ε 就照常贪婪；小于 ε 就闭眼乱选一个（乱选也可能恰好选中贪婪动作，所以贪婪动作的总概率是 1−ε+ε/|𝒜|）。 ε=0 退化为纯贪婪，ε=1 变成纯随机。无论 ε 多大，贪婪动作的概率永远不低于其他动作——「主见」始终在，只是不再固执。

互动 3-Aε-greedy 的概率天平

假设在某状态，贪婪动作是「→」。拖动 ε，看五个动作的概率如何此消彼长；点「抽样验证」用 10000 次模拟抽样对照理论值。

注意两个端点：ε=0 时贪婪动作独占 100%；ε=1 时五个动作各 20%——完全随机。 ε=0.5 时贪婪动作仍有 60%，「主见」还是明显的。

3.4MC ε-Greedy 算法

把 ε-greedy 装进上一章的 MC Exploring Starts，只需改一处：策略改进时不再输出纯贪婪策略，而是输出 ε-greedy 策略 （把式 (3.1) 套在 argmax 动作上）。出发点随便选、首个动作不再强制随机——探索交给策略自己。

📋 算法：MC ε-Greedy（原书算法 5.3，every-visit 版）
对每条回合：
  1. 从任意 (s₀, a₀) 出发（无需探索性出发），按当前 ε-greedy 策略走 T 步；
  2. g ← 0，从末尾倒推 t = T−1, …, 0：g ← γg + rt+1；更新 Returns、q(st,at)；
  3. 改进：a* = argmaxa q(st,a)，π(·|st) 按式 (3.1) 设为 ε-greedy。
互动 3-C现场训练 MC ε-Greedy（出发点永远固定在左上角！）

每回合从左上角出发走 1000 步——只有这一个出发点，探索全靠 ε。 左图显示 q 导出的贪婪策略与价值；右侧曲线照例是「贪婪策略与最优的真实差距」。

实验脚本：① 默认 ε=0.5 训练 100 回合：尽管出发点从未变过，差距照样落到 0 附近—— 对照互动 2-C 实验②（贪婪策略 + 固定出发点 = 卡死），体会软性策略立了多大的功； ② 重置，把 ε 拖到 0.05 再训练 100 回合：常会卡在次优——探索太少，回到了老毛病； ③ 重置，ε=0.5 且勾选「衰减」：先大胆探索、后专心利用，这是实战中最常用的日程表（GLIE）。

3.5天下没有免费的探索：ε 的代价

ε-greedy 解决了探索，但别忘了：最后执行任务的策略也是软的，它每一步都有 ε 的概率「抽风」。 探索是有价格的，价格挂在价值函数上。下面这个实验对应原书图 5.6/5.7，把价格算给你看。

互动 3-D给探索定价：ε-greedy 策略的真实价值

取最优贪婪策略，把它「软化」成 ε-greedy（主见方向不变），用模型精确评估其真实价值。 勾选「在 ε-greedy 家族内重新寻优」后，会另外算出该 ε 下最好的 ε-greedy 策略，看它的主见是否还和贪婪最优一致。

箭头长度 ∝ 概率。注意观察目标格（蓝绿色）： ε 大到一定程度后，重新寻优的策略会让它「逃离目标」——因为呆在原地时，ε 的抽风迟早把它甩进周围 −10 的禁区，不如跑路。

实验脚本：① 把 ε 从 0 拖到 0.5，看所有格子的价值一路下沉，目标格跌得最惨（它被禁区包围）； ② 勾选「重新寻优」：ε=0.1 时最优 ε-greedy 策略的主见仍与贪婪最优完全一致（一致性 25/25）； ε≈0.2 起开始变形；ε=0.5 时目标格干脆选择出逃。结论：想让学到的策略既能探索又不变形，ε 要够小； 或者训练时用大 ε、部署时把 ε 摘掉。

3.6本章小结与下一站

  • 探索性出发不现实；病根是贪婪策略的确定性，药方是软性策略。
  • ε-greedy：以 1−ε 守主见、以 ε 闭眼乱选。改进步骤里只需把「纯贪婪」换成「ε-greedy」。
  • ε 大 → 探索均匀、样本扎实，但策略本身被探索拖累（价值下沉、甚至主见变形）； ε 小 → 策略接近最优，但探索偏食。实战折中：ε 先大后小（衰减）。
  • 三章连起来看：MC Basic（思想原型）→ MC ES（榨干样本 + 即时改进）→ MC ε-Greedy（自带探索）。 每一步都只改动一个部件，这种「换零件」式的演化是读懂 RL 算法家族的正确姿势。
🤔 带走的思考题
  1. 互动 3-B 中 ε=0.5 时访问分布严重不均，但理论上仍保证「每个 (s,a) 被访问无穷多次」。为什么「无穷多次」还不够，我们还在乎均匀性？（提示：同样的总步数预算下，方差花在哪了？）
  2. 训练时用 ε-greedy 收集数据、评估改进的也是这个 ε-greedy 策略，这叫同策略（on-policy）学习。能否「用 ε=1 的策略闯荡、却学习纯贪婪策略的价值」？这就是异策略（off-policy）的问题——Q-learning 的伏笔。
  3. MC 永远要等一条（很长的）回合走完才能更新。能不能走一步就学一步？——能。那就是时序差分（TD）学习： 把「实际回报 G」换成「一步奖励 + 对后面的估计」。建议下一步阅读原书第 7 章，配合 demo 仓库的 td_prediction.ipynb 与 SARSA_demo.ipynb。
