第 2 章从评估到改进：MC Basic 与 Exploring Starts

第 1 章学会了给策略「打分」（预测）。这一章解决控制（control）问题： 不断利用打分结果改进策略，直到找到最优策略——而且全程不碰模型。

2.1复习：策略迭代这台发动机

回忆一下（有模型的）策略迭代，它是一台两冲程发动机，反复做两件事：

  1. 策略评估：算出当前策略 πk 的价值；
  2. 策略改进：在每个状态贪婪地换上更好的动作， πk+1(s) = argmaxa qπk(s, a)。

注意改进这一步用的是动作价值 q(s,a)，不是 V(s)。 这正是整台发动机的心脏：要选动作，就必须知道「在 s 做 a 到底值多少」。算 q 有两条路：

  • 有模型：qπ(s,a) = Σr p(r|s,a)·r + γ Σs′ p(s′|s,a)·Vπ(s′)。要用到转移概率——没模型就此路不通。
  • 免模型：qπ(s,a) 的定义本来就是期望回报 𝔼[Gt | St=s, At=a]。 是期望？那就采样取平均！从 (s,a) 出发按 π 跑很多回合，平均回报就是 q 的估计——这正是第 1 章练好的功夫。
💡 一句话顿悟
把策略迭代里「用模型算 q」整块抠下来，换成「用采样估 q」，其余一个字不改——你就发明了第一个免模型强化学习算法。 就这么简单。

2.2MC Basic：最朴素的免模型控制

📋 算法：MC Basic（《强化学习的数学原理》算法 5.1）
从任意初始策略 π₀ 出发，第 k 轮（k = 0, 1, 2, …）做：
  1. 策略评估：对每个状态-动作对 (s, a)，从 (s,a) 出发按 πk 收集足够多回合， 用平均回报近似 qπk(s, a)；
  2. 策略改进：每个状态换上贪婪动作 a*k(s) = argmaxa qk(s, a)。

本章换一块更有意思的场地：原书的 5×5 网格世界。蓝绿色格子是目标（进入得 +1）， 橙色是禁区（进入得 −10，但允许进入），撞边界弹回得 −1，其余移动得 0。动作有五个：上、右、下、左、原地不动。 注意这个任务没有终点——到达目标后回合不结束，最优解是「走到目标然后原地呆着刷分」。 没有终点，第 1 章的思考题就来了：无限长的回报怎么采样？答案：截断。 每个回合只跑 L 步，回报只累计这 L 步。那么 L 该取多大？这就是下面实验的主角。

互动 2-A回合长度 L 如何决定成败（复现原书图 5.4）

下面现场运行 MC Basic 直到策略稳定（环境与策略都确定，所以每个 (s,a) 采一条长度 L 的回合就够了）。 左图：最终的状态价值 v(s) = maxa q(s,a) 与贪婪策略箭头。拖动 L，观察价值如何从目标点「往外生长」。

曲线：非零价值的状态个数 随 L 的变化（自动扫描时绘制）。

观察清单：① L=1 时只有目标四邻有非零价值——回合太短，远处的状态根本「看不到」奖励； ② L 逐渐加大，价值像涟漪一样从目标向外扩散，离目标越近的状态越早「苏醒」； ③ 本图中从最远的左下角到达目标需要约 15 步，所以 L ≥ 15 后所有状态才有非零估计； ④ L=30 左右策略已经最优（看箭头），虽然价值数字还偏小——找对策略不需要把价值算准。

⚠ 这里藏着一个大问题：稀疏奖励
这个网格里只有目标给正奖励——不摸到目标，回报全是 0 或负数，学习就无从谈起。这叫稀疏奖励（sparse reward）。 状态空间一大，「碰巧走到目标」难如登天，这是强化学习的核心难题之一。 常见解药是把奖励铺得密一点（比如目标附近也给点小奖励，形成「引力场」）。 你会在编程实践④里亲手体验奖励设计（step cost）对学到的策略的塑造力。

2.3把一条轨迹「榨干」：initial / first / every-visit

MC Basic 思想干净，但奢侈得离谱：每个 (s,a) 都要专门收集一批回合，回合里后面经过的大量 (s,a) 全被扔掉了。 看看一条轨迹里到底藏着多少财富：

s₁ →a₂→ s₂ →a₄→ s₁ →a₂→ s₂ →a₃→ s₅ →a₁→ ⋯

轨迹每经过一个 (s,a)，称为对它的一次访问（visit）。从任何一次访问处把轨迹「剪开」， 后半段就是一条从该 (s,a) 出发的子回合——一个免费的回报样本！利用方式有三档：

  • initial-visit：只用开头那个 (s,a)（MC Basic 的用法，最浪费）；
  • first-visit：每个 (s,a) 的首次访问各贡献一个样本；
  • every-visit：每次访问都贡献样本（最省，样本间有相关性，但依然收敛）。

数一数：every-visit 口径下，一条 12 步的轨迹贡献 12 个样本——是 initial-visit 的 12 倍。 样本利用率的差距随轨迹变长越拉越大，这就是为什么实用的 MC 算法都抛弃了 initial-visit。

2.4MC Exploring Starts：高效版 MC Basic

把 2.3 的「榨干技巧」装回 MC Basic，再加一个提速点——不等估计收集完美就改进策略： 每跑完一条回合就立刻用它更新 q、立刻贪婪改进。单条回合的估计当然很糙， 但「评估不精确也照样改进」正是广义策略迭代（GPI）的精髓，后面几乎所有算法都这么干。

📋 算法：MC Exploring Starts（原书算法 5.2，every-visit 版）
对每条回合：
  1. 探索性出发：随机选起点 (s₀, a₀)，保证每个状态-动作对都可能被选中；之后按当前策略走 T 步；
  2. 令 g ← 0，从轨迹末尾往回遍历 t = T−1, …, 0：
    • g ← γ·g + rt+1（倒带）；
    • Returns(st, at) 累加 g，q(st, at) ← 平均值；
    • 立刻改进：π(st) ← argmaxa q(st, a)。

名字里的探索性出发（exploring starts）是算法的命门： 必须保证每个 (s,a) 都有机会作为起点被反复尝试。为什么？贪婪策略是「独食」的—— 一旦它认定某动作差（哪怕只是因为还没试过），就再也不会选它，对应的 q 永远得不到更新，错判将无人纠正。 探索性出发用「行政手段」强行保证人人有份。它管不管用？没有它会怎样？做实验。

初始策略完全随机。每条回合 100 步。左图实时显示当前贪婪策略与 v(s) = maxa q(s,a)； 右侧曲线是「当前策略与最优策略的真实差距」（用模型精确评估，只当裁判）。

| |

差距 = (v*(s) − vπ(s)) 对所有 s 取平均；降到 0 即策略最优。

实验脚本：① 开着探索性出发训练 1000 回合——差距曲线落到 0，箭头全部指向目标，(s,a) 覆盖率接近 100%； ② 重置，关掉探索性出发（每回合都从左上角出发）再训练 1000 回合——打开「访问热力」看看： 智能体很快锁死在一条自以为是的路线上，大片 (s,a) 从未尝试，差距曲线卡在半空再也下不去。 这就是「探索不足」的活体标本；③ 反复按「训练 1 回合」单步观察：策略箭头是一格一格被「点亮」的， 顺序大致从目标附近向外蔓延——和互动 2-A 的涟漪是同一个道理。

2.5本章小结

  • 免模型控制 = 策略迭代 + 「用采样平均估 q」。最朴素的实现是 MC Basic。
  • 回报截断长度 L 必须足以从最远的状态摸到奖励；价值总是从奖励源向外生长。稀疏奖励因此致命。
  • 一条轨迹的每次访问都是一个免费样本：every-visit + 倒带累加是标准写法。
  • MC Exploring Starts = MC Basic + 榨干轨迹 + 单回合即时改进（GPI）。
  • 贪婪改进会扼杀探索，必须靠「探索性出发」续命——但现实中（机器人、真实交互）你往往没法把系统拽到任意 (s,a) 重启。怎么办？下一章给出优雅的答案。
